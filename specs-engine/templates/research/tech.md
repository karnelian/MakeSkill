### 4. tech.md 출력 구조

```markdown
# Tech Research

## 요약

## 기술 영역별 비교
### 영역 N: {이름}
| 항목 | 옵션A | 옵션B | 옵션C |
| GitHub Stars / npm 다운로드 / 성능 / 번들 크기 / 학습 곡선 / 생태계 / 채용 시장 / 대표 사례 / 라이선스 |

## 아키텍처 패턴
### 패턴 N: {이름}
- 설명 / 적합한 경우 / 사례 [출처] / 장점 / 단점

## CTO 추천 스택
| 영역 | 추천 | 이유 |

## 기술 리스크
| 리스크 | 가능성 | 영향 | 완화 방안 |

## 리서치 한계

## 출처율 검증
- [출처] 수 / [추정] 수 / 출처율
```

### 5. 출처율 자체검증

1. `[출처:` 패턴 수 집계
2. `[추정]` 패턴 수 집계
3. 출처율 = `[출처] / ([출처] + [추정]) x 100`
4. 90% 미만 시 추가 WebSearch (최대 3회)
5. 여전히 미달이면 `## 리서치 한계`에 사유 명시
