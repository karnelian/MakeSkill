## 프로세스

### 1. seed.md에서 컨텍스트 추출

- 아이디어, 유형, 타겟

### 2. 경쟁사 식별 (WebSearch)

- **직접 경쟁사**: 같은 문제를 같은 방식으로 풂 (3~5개)
- **간접 경쟁사**: 같은 문제를 다른 방식으로 풂 (3~5개)
- **대안**: 사용자가 현재 사용하는 것 (엑셀, 노트, 수동 등)

검색 키워드:
- "{카테고리} best tools 2024"
- "{문제} solution alternatives"
- "{경쟁사이름} vs competitors"
- "{카테고리} market share"

### 3. 개별 경쟁사 심층 조사

각 경쟁사에 대해 조사:

| 항목 | 조사 내용 |
|------|----------|
| 기본 정보 | 설립연도, 본사, 직원 수 |
| 제품 | 핵심 기능, 지원 플랫폼 |
| 가격 | 무료/유료 플랜, 가격대 |
| 사용자 | MAU/DAU, 다운로드 수, 리뷰 수 |
| 펀딩 | 투자 라운드, 총 투자액, 밸류에이션 |
| 강점 | 차별화 포인트 |
| 약점 | 사용자 불만, 부족한 점 |

### 4. competitor.md 출력 구조

```markdown
# Competitor Research

## 경쟁 환경 요약

## 직접 경쟁사 (각: 이름/설립/제품/가격/사용자/펀딩/강점/약점)

## 간접 경쟁사

## 기능 비교 매트릭스
| 기능 | 우리 (계획) | 경쟁사A | 경쟁사B | 경쟁사C |

## 가격 비교
| 서비스 | 무료 플랜 | 기본 | 프로 | 엔터프라이즈 |

## 차별화 기회 (최소 2개)
### 기회 N: {설명}
- 현재 시장 상태 / 사용자 불만 / 우리의 접근 / 난이도

## 경쟁 포지셔닝 맵

## 리서치 한계

## 출처율 검증
- [출처] 수 / [추정] 수 / 출처율
```

### 5. 출처율 자체검증

1. `[출처:` 패턴 수 집계
2. `[추정]` 패턴 수 집계
3. 출처율 = `[출처] / ([출처] + [추정]) x 100`
4. 90% 미만 시 추가 WebSearch (최대 3회)
5. 여전히 미달이면 `## 리서치 한계`에 사유 명시
