## 프로세스

### 1. seed.md에서 컨텍스트 추출

- 타겟 사용자 힌트 (인텐트에서), 프로젝트 유형

### 2. 타겟 사용자 인구통계 (WebSearch)

| 항목 | 검색 키워드 예시 |
|------|----------------|
| 규모 | "{타겟 그룹} population statistics 2024" |
| 연령대 | "{서비스 카테고리} user demographics age" |
| 지역 | "{서비스 카테고리} users by country region" |
| 소득 | "{타겟 그룹} average income spending" |
| 디바이스 | "{타겟 그룹} device usage mobile desktop" |

### 3. 행동 패턴 조사 (WebSearch)

| 항목 | 검색 키워드 예시 |
|------|----------------|
| 사용 맥락 | "when do people use {카테고리} app" |
| 사용 빈도 | "{카테고리} app usage frequency daily weekly" |
| 체류 시간 | "{카테고리} average session time" |
| 결제 패턴 | "{카테고리} willingness to pay survey" |
| 이탈 원인 | "{카테고리} app churn reasons why users leave" |

### 4. 페인 포인트 & 니즈 발굴

조사 소스: Reddit/커뮤니티 포럼, 앱스토어 리뷰 (별점 낮은 것), 경쟁사 리뷰, 서베이 리포트

### 5. 페르소나 생성 (2~3개)

각 페르소나: 이름, 나이, 직업, 기술 수준, 목표, 좌절, 현재 해결 방법, 기대, 결제 의향

### 6. 사용자 여정 맵 초안

| 단계 | 행동 | 감정 | 접점 | 기회 |
|------|------|------|------|------|
| 인지 / 탐색 / 시작 / 사용 / 유지 / 추천 |

### 7. user.md 출력 구조

```markdown
# User Research

## 타겟 사용자 요약

## 인구통계
| 항목 | 데이터 | 출처 |

## 행동 패턴
| 항목 | 데이터 | 출처 |

## 페인 포인트
| 순위 | 페인 포인트 | 빈도 | 출처 |

## 페르소나 (2~3개)

## 사용자 여정 맵

## UX 리서처 인사이트

## 리서치 한계

## 출처율 검증
- [출처] 수 / [추정] 수 / 출처율
```

### 8. 출처율 자체검증

1. `[출처:` 패턴 수 집계
2. `[추정]` 패턴 수 집계
3. 출처율 = `[출처] / ([출처] + [추정]) x 100`
4. 90% 미만 시 추가 WebSearch (최대 3회)
5. 여전히 미달이면 `## 리서치 한계`에 사유 명시
