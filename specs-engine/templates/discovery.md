## 프로세스

### 1. 리서치 종합

4개 리서치 파일을 읽고 핵심 발견을 요약:
- 시장 핵심: TAM, CAGR, 핵심 트렌드
- 경쟁 핵심: 주요 경쟁사, 차별화 기회
- 기술 핵심: CTO 추천, 기술 리스크
- 사용자 핵심: 핵심 페인 포인트, 가장 유력한 페르소나

### 2. Gray Area 식별

리서치 결과에서 명확하지 않은 영역을 식별:

| 카테고리 | 예시 |
|----------|------|
| 비주얼/디자인 | 아트 스타일 방향, UI 톤앤매너 |
| 기술 트레이드오프 | 성능 vs 개발 속도, 네이티브 vs 크로스플랫폼 |
| 비즈니스 모델 | 무료+광고 vs 프리미엄, 구독 주기 |
| 콘텐츠/기능 | MVP 범위, 우선순위, 핵심 기능 정의 |
| 리소스 | 아트/사운드 확보 방법, 외주 vs 자체 |

각 Gray Area에 결정 필요 등급 부여: S급(사용자 결정) / A급(AI CEO 결정) / B급(담당 역할 결정)

### 3. 역할별 토론

각 S급 결정에 대해 토론 형식으로 진행:

```
| 역할 | 주장 | 근거 |
|------|------|------|
| CTO | A안 | ... |
| CFO | B안 | ... |
| 마케터 | A안 | ... |
| UX 리서처 | B안 | ... |

핵심 쟁점: ...
AI CEO 추천: X안 (이유)
```

### 4. S급 결정 수집

사용자에게 한 번에 3~5개씩 S급 결정을 확인받는다.

공통 S급 결정:
1. 개발환경 / 기술 스택
2. 타겟 사용자 (최종 확정)
3. 핵심 차별점
4. 수익 모델
5. 아트 리소스 전략
6. 사운드 리소스 전략

유형별 추가 S급 결정:
- 게임: 코어루프 방향, 과금 모델
- 웹 SaaS: B2B vs B2C, 셀프서브 vs 세일즈
- 모바일: 네이티브 vs 크로스플랫폼, 오프라인 범위
- API: REST vs GraphQL, 과금 체계
- 크롬 확장: 프리미엄 모델, 백엔드 여부
- 데스크톱: Electron vs Tauri, 배포 채널

### 5. A급 결정 (AI CEO 판정)

S급이 아닌 중요 결정은 AI CEO가 판정하고 보고한다.

### 6. Locked Decisions 기록

모든 결정을 확정 목록으로 기록. 이후 Phase에서 이 결정에 반하는 내용을 작성하지 않는다.

### 7. context.md 생성

아래 구조를 반드시 준수한다:

```markdown
# Project Context

> {프로젝트명} — {한줄 설명}
> 작성일: {날짜}

---

## §1 프로젝트 기본 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | {프로젝트명} |
| 유형 | {game|web-saas|mobile|desktop|chrome-ext|api-service} |
| 타겟 | {seed.md 인텐트에서 추출} |
| 운영 모드 | {1인 모드|팀 모드|회사 모드} |
| 타임라인 | {기간} |

## § Software Stack
- 프론트엔드: ...
- 백엔드: ...
- DB: ...
- 호스팅: ...
- (유형별 추가)

## § Locked Decisions
### S급 (사용자 결정)
- [S1] ...
- [S2] ...

### A급 (AI CEO 결정)
- [A1] ...
- [A2] ...

## § 핵심 수치 SSOT
- TAM: $XXB
- SAM: $XXB
- SOM: $XXM (Y1)
- CAGR: X.X%
- Y1 MAU 목표: ...
- Y1 전환율: ...
- Y1 MRR: ...
- Y1 ARR: ...

## § Gray Areas
| # | 영역 | 설명 | 결정 등급 | 결정 결과 |

## § 마일스톤 개요
| 마일스톤 | 목표 | 기간 | 핵심 산출물 |

## § 역할 배정
- 공통: CEO, 마케터, UX 리서처, CTO, CFO
- 전문가: (유형별)

## Cross-validation Check
| 검증 항목 | 계산 | 결과 | 일치 |
```

### 8. SSOT 산술 교차검증

context.md 생성 직후 반드시 검증:
1. 매출 역산: `MAU x 전환율(%) x 가격` = 월 매출
2. 연 매출 역산: `월 매출 x 12` = 연 매출
3. 복수 플랜 합산: 각 플랜별 역산 후 합산 = 총 매출
4. DAU/MAU 비율: 전 Year 일관성 확인

FAIL 시: S급 결정 값이 앵커 -> 나머지를 역산 수정
