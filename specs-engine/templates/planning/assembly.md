> 12개 챕터를 합본하여 최종 기획서를 생성하고, 분량/정합성을 검증한다.

---

## 읽어야 할 파일

| 파일 | 필수 | 부재 시 행동 |
|------|:----:|-------------|
| `specs/{프로젝트명}/planning/_contract.md` | 🔴 | **BLOCK** — 전체 검증 기준 없이 조립 불가 |
| `specs/{프로젝트명}/skeleton.md` | 🔴 | **BLOCK** — 목표 분량 기준 없이 검증 불가 |
| `specs/{프로젝트명}/planning/01~12-*.md` 전체 12개 | 🔴 | **BLOCK** — 누락 챕터 있으면 합본 불가. 누락 챕터 Phase부터 재실행 |

## 생성할 파일

- `specs/{프로젝트명}/{프로젝트명}-기획서.md`

## 조립 절차

### Step 1: 챕터별 분량 검증

각 챕터의 `wc -l` 결과를 skeleton 최소 기준과 대조한다.

| 챕터 | 최소 | 목표 | 실제 | 판정 |
|------|------|------|------|------|
| Ch01 | 80 | 130 | ? | |
| Ch02 | 120 | 200 | ? | |
| Ch03 | 120 | 200 | ? | |
| Ch04 | 120 | 200 | ? | |
| Ch05 | 250 | 400 | ? | |
| Ch06 | 140 | 230 | ? | |
| Ch07 | 120 | 200 | ? | |
| Ch08 | 80 | 140 | ? | |
| Ch09 | 80 | 140 | ? | |
| Ch10 | 70 | 120 | ? | |
| Ch11 | 70 | 120 | ? | |
| Ch12 | 50 | 100 | ? | |
| **합계** | **1,300** | **2,180** | **?** | |

미달 챕터가 있으면 표 추가, 상세 설명 확대, 사례 추가로 보강 후 재측정.

### Step 2: 숫자 교차검증

Ch09 월별 KPI 테이블을 canonical source로 지정하고, 전 챕터와 대조한다.

- Ch09에서 M12 행의 MAU, DAU, 유료 구독자, MRR 추출
- Year 1 누적 = M1~M12 MRR 합산 (단순 평균 x 12 금지)
- M12 연환산 = M12 MRR x 12
- Ch01/Ch02/Ch07과 대조. 불일치 시 Ch09 기준으로 수정

### Step 3: Contract 기반 정합성 검증

| # | 검증 항목 | 대상 | 방법 |
|---|----------|------|------|
| 1 | 프리미엄 비교표 (C3) | Ch07, Ch01 | 셀 단위 1:1 대조 |
| 2 | v1/v2 기능 경계 (C4) | Ch03, Ch06, Ch07 | v2 기능이 v1 맥락에 사용되면 FAIL |
| 3 | 마일스톤 라벨 (C5) | 전체 | 이름/기간 일치 대조 |
| 4 | 기술 스택 (C6) | Ch06 | 스택명 대조 |
| 5 | 핵심 수치 (C2) | Ch01, Ch02, Ch07, Ch09 | 동일 수치 대조 |
| 6 | 가격 (C1) | Ch02, Ch07, Ch09 | 가격 대조 |
| 7 | 내부 메모 잔존 | 전체 | "확정 후 정합", "TODO", "TBD" Grep → 0건 |

FAIL 시 contract 기준으로 수정 후 재검증 (최대 3회).

### Step 4: 합본

```markdown
# {프로젝트명} 기획서

> 생성일: {날짜}
> 버전: v1.0

---

## 목차
1. Executive Summary
2. 시장 분석
...
12. 부록

---

(각 챕터 내용을 순서대로 합본)
```

**합본 규칙**:
- 헤딩 레벨 정규화: 각 챕터의 최상위 헤딩을 `##`로 통일
- 챕터 간 `---` 페이지 구분선 삽입
- 개별 챕터 파일의 메타데이터(버전, 날짜 등) 제거

### Step 5: 최종 메타 정보

기획서 마지막에 문서 정보 표를 추가한다:
총 라인 수, 생성일, 프로젝트 유형, 참여 역할, [출처] 수, [추정] 수 및 비율.
