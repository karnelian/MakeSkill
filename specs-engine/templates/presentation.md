## 생성할 파일

생성 디렉토리: `specs/{프로젝트명}/presentation/`

### 1. slides-content.md — 슬라이드별 텍스트 콘텐츠

각 슬라이드: 제목, 헤드라인, 본문, 차트 유무, 발표자 노트

### 2. generate-ppt.js — pptxgenjs 실행 파일

```javascript
const pptxgen = require("pptxgenjs");
const pres = new pptxgen();
// 마스터 슬라이드 + 슬라이드별 생성
pres.writeFile({ fileName: "{프로젝트명}-deck.pptx" });
```

실행: `npm install pptxgenjs && node generate-ppt.js`

## 용도별 프리셋

### investor (투자 유치) — 12~15장

1. 표지 / 2. 문제 정의 / 3. 솔루션 / 4. 시장 규모 (TAM/SAM/SOM) / 5. 비즈니스 모델 / 6. 경쟁 우위 / 7. 제품 데모 (UI 목업) / 8. 트랙션 / 9. GTM / 10. 재무 계획 (3년) / 11. 팀 / 12. 투자 요청 / 13. 연락처

### internal (내부 보고) — 8~12장

1. 표지 / 2. 프로젝트 개요 / 3. 시장 분석 / 4. 제품 정의 / 5. 기술 스택 / 6. 로드맵 / 7. 리소스 & 예산 / 8. 리스크 / 9. KPI / 10. Q&A

### store (스토어 소개) — 5~8장

1. 로고 + 이름 / 2. 핵심 가치 / 3. 주요 기능 (스크린샷) / 4. 사용 흐름 / 5. 리뷰/평점 / 6. 다운로드 CTA

### partnership (파트너십 제안) — 10~12장

1. 표지 / 2. 회사 소개 / 3. 시장 기회 / 4. 파트너십 가치 / 5. 협력 모델 / 6. 시너지 / 7. 실행 계획 / 8. 성공 사례 / 9. 기대 효과 / 10. 연락처

## 필수 UI 화면 목업 슬라이드 규칙

모든 프리셋에서 **제품 데모/주요 기능 슬라이드**에 적용:

- **모든 주요 화면** 포함 (핵심 1~2개만 넣지 않음)
- 디바이스 프레임 안에 렌더링 (폰/브라우저/데스크톱 윈도우)
- UI 요소는 **모양/색/크기** 시각 표현 (CSS gradient, box-shadow 등)
- **텍스트 플레이스홀더 절대 금지**: `[버튼]`, `[HP바]` 등 텍스트 대체 불가
- 인터랙션 상태 1개 이상 표현 (활성/비활성/로딩/호버)

### 유형별 필수 화면

| 유형 | 필수 화면 |
|------|----------|
| **게임** | 로비, 인게임(HUD), 팝업(레벨업/상점), 결과, 코어루프 다이어그램 |
| **모바일 앱** | 온보딩, 홈, 상세, 프로필, 스크린 플로우 |
| **웹 SaaS** | 랜딩, 대시보드, 편집, 가격 페이지, 전환 퍼널 |
| **API 서비스** | 개발자 포털, API 레퍼런스, 아키텍처 다이어그램 |
| **크롬 확장** | 팝업, 사이드패널, 오버레이 |
| **데스크톱 앱** | 메인 윈도우, 설정, 트레이 |

## SSOT 규칙

### Rule 1: 마일스톤 범위 = SSOT 그대로

- M1~M4 기능 범위/콘텐츠 수량/기간 = planning/05-product-definition.md §5.3, §5.5 원문
- M2 이후 기능을 M1에 포함 금지

### Rule 2: 시나리오 수치 = 기획서 원문

- Conservative/Base/Optimistic 정의(MAU%, ARPU%) + 연매출 = 기획서 재무 섹션 직접 복사
- 단순 배율(60%, 130%) 적용 금지
- 4년 누적 순이익 = 기획서 원문 (허용 오차 ±$50K)
- 기획서가 분산 구조일 경우 context.md §4 수치를 SSOT로 사용

### Rule 3: 비용/손익 = 기획서 프레임워크 동일

- 총 비용/영업이익/순이익 항목 구성 = 기획서 손익계산서 그대로
- 스토어 수수료 포함/제외 방식 = 기획서와 동일
- Y1 총 비용 ↔ 기획서 ±5% 이내
- 기획서가 분산 구조일 경우 context.md §4 + §5 수치를 SSOT로 사용
