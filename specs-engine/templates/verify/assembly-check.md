# Template: 합본 검증 (Assembly Check)

## 읽어야 할 파일

| 파일 | 필수 | 부재 시 행동 |
|------|:----:|-------------|
| `specs/{프로젝트명}/skeleton.md` | 🔴 | **BLOCK** — 목표 라인수 기준 없이 분량 검증 불가 |
| `specs/{프로젝트명}/{프로젝트명}-기획서.md` | 🔴 | **BLOCK** — 합본 파일 없으면 검증 대상 없음 |
| `specs/{프로젝트명}/planning/*.md` 개별 챕터 | 🟡 | **DEGRADE** — 합본과 개별 챕터 비교 생략, 합본 단독 검증 |

## 실행할 패턴

### P-ASM-1: 챕터별 라인수 vs skeleton

- **검증**: 합본 파일에서 각 챕터(# 1장 ~ # 12장) 구간의 라인수 계산 → skeleton.md 목표와 대조
- **기준**: 각 챕터 목표의 80% 이상 → PASS, 80% 미만 → FAIL
- **결과 형식**:

```
| 챕터 | skeleton 목표 | 합본 실제 | 달성률 | 판정 |
|------|-------------|----------|--------|------|
| 1장 | 120 | 115 | 96% | PASS |
| 5장 | 400 | 280 | 70% | FAIL |
```

- **FAIL 시**: 해당 챕터의 부족 섹션 식별 → 보강 지시

### P-ASM-2: 구조 검증

- **검증 항목**:
  1. 합본 파일에 ch01~ch12 **전체 12개 챕터** 존재 확인
  2. 각 챕터의 **필수 섹션**(skeleton.md 정의) 존재 확인
  3. 합본 파일 **총 라인수** ≥ 1,300 확인
  4. 챕터 간 **순서** 정확성 (1→2→...→12)
  5. 빈 챕터(라인수 10 미만) 감지 → FAIL
- **기준**: 전체 구조 정합 → PASS
