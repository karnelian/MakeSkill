## 3. 유형별 에이전트 역할 매핑 (v5.4)

> 동일한 템플릿(ux.md, data-api.md 등)을 사용하되,
> 유형에 따라 에이전트의 **역할 이름, 관점, 핵심 산출물**이 자동 변환된다.
> runner.js가 context.md 유형을 읽어 해당 매핑을 프롬프트에 주입한다.

### 공통 템플릿 → 유형별 역할 변환

| 템플릿 | 게임 | 웹 SaaS | 모바일 앱 | API 서비스 | 데스크톱 앱 | 크롬 확장 |
|--------|------|---------|----------|-----------|-----------|----------|
| **ux.md** | 게임 UI/HUD 설계자 | UI/UX 디자이너 | 모바일 UX 디자이너 | — (스킵) | 데스크톱 UX 디자이너 | 확장 UX 디자이너 |
| **data-api.md** | 게임서버/네트워크 설계자 | 데이터 아키텍트 + API 설계자 | 백엔드 설계자 | API 설계자 + 데이터 아키텍트 | 시스템 설계자 | 웹 플랫폼 전문가 |
| **security-infra-qa.md** | 안티치트/서버권위 + 인프라 + QA | 보안 + DevOps + QA | 보안 + 성능 + QA | 보안 + 인프라 + QA | 크로스플랫폼 + 배포 + QA | 프라이버시 + QA |
| **impl/dev-env.md** | 게임 클라이언트 환경 설정 | 프론트+백엔드 개발환경 | 모바일 개발환경 | API 서버 개발환경 | 데스크톱 빌드 환경 | 확장 개발환경 |

### 에이전트 프롬프트 주입 형식

Verify/Design/Implementation Phase에서 에이전트 프롬프트 상단에 주입:

```
## 프로젝트 유형 컨텍스트
유형: {게임}
당신의 역할: {게임 서버/네트워크 설계자}
이 프로젝트에서 "백엔드"는 "게임 서버"를 의미합니다.
이 프로젝트에서 "API"는 "클라이언트-서버 프로토콜"을 의미합니다.
이 프로젝트에서 "인증"은 "플레이어 세션 관리"를 의미합니다.
이 프로젝트에서 "보안"은 "안티치트 + 서버 권위"를 의미합니다.
```

## 4. 유형별 설계서 디렉토리 구조

```
design/
├── ux/                  # 공통: 화면 목업, 플로우
├── data/                # 공통: DB 스키마, ERD
├── api/                 # 공통: API 엔드포인트
├── security/            # 공통: 인증/인가, 보안
├── infra/               # 공통: 호스팅, CI/CD
├── qa/                  # 공통: 테스트 전략
└── {유형별}/
    ├── game-design/     # 게임: 코어루프, 밸런스, 이코노미
    ├── growth/          # 웹SaaS: 퍼널, 온보딩
    ├── mobile/          # 모바일: 플랫폼별, 오프라인
    ├── developer/       # API: SDK, 문서, 온보딩
    ├── extension/       # 크롬: 매니페스트, 권한
    └── desktop/         # 데스크톱: OS 통합, 업데이트
```

## 4. 유형별 마일스톤 차이

### 게임
| 마일스톤 | 목표 |
|----------|------|
| M1 | 프로토타입 (코어루프 검증) |
| M2 | 알파 (핵심 콘텐츠 + 밸런스) |
| M3 | 베타 (전체 콘텐츠 + 폴리시) |
| M4 | 출시 (최적화 + QA + 스토어 등록) |

### 웹 SaaS
| 마일스톤 | 목표 |
|----------|------|
| M1 | MVP (핵심 기능 + 인증) |
| M2 | 베타 (전체 기능 + 결제) |
| M3 | 퍼블릭 (성능 최적화 + 온보딩) |
| M4 | 성장 (분석 + A/B 테스트 + 확장) |

### 모바일 앱
| 마일스톤 | 목표 |
|----------|------|
| M1 | MVP (핵심 플로우 + 오프라인) |
| M2 | 베타 (전체 기능 + 푸시) |
| M3 | 스토어 심사 (최적화 + 가이드라인 준수) |
| M4 | 출시 (ASO + 모니터링) |

### API 서비스
| 마일스톤 | 목표 |
|----------|------|
| M1 | 알파 (핵심 엔드포인트 + 인증) |
| M2 | 베타 (전체 API + SDK + 문서) |
| M3 | 퍼블릭 (Rate Limit + 모니터링) |
| M4 | GA (SLA + 과금 + 대시보드) |

### 크롬 확장
| 마일스톤 | 목표 |
|----------|------|
| M1 | MVP (핵심 기능 + Manifest V3) |
| M2 | 베타 (전체 기능 + 설정) |
| M3 | 스토어 심사 (권한 최소화 + 프라이버시) |
| M4 | 출시 (ASO + 리뷰 관리) |

### 데스크톱 앱
| 마일스톤 | 목표 |
|----------|------|
| M1 | MVP (핵심 기능 + 단일 OS) |
| M2 | 베타 (전체 기능 + 크로스플랫폼) |
| M3 | RC (자동 업데이트 + 코드 서명) |
| M4 | 출시 (배포 채널 + 모니터링) |
