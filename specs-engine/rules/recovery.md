# Recovery — 에러 복구 규칙

모든 Phase에서 발생 가능한 장애 상황에 대한 복구 전략.

---

## 1. 장애 유형 분류

| 유형 | 설명 | 예시 |
|------|------|------|
| **F1: 필수 파일 부재** | 이전 Phase 산출물이 없음 | context.md 미생성 상태에서 Planning 시도 |
| **F2: 에이전트 중단** | 컨텍스트 초과, 타임아웃, 실행 오류 | 긴 챕터 작성 중 잘림 |
| **F3: 검증 반복 실패** | 자동 수정 3회 후에도 FAIL | P1 수치 정합성이 구조적으로 불가 |
| **F4: S급 결정 미응답** | 사용자 결정 대기 중 다른 Phase 진행 시도 | 기술 스택 미확정 상태에서 설계 시도 |
| **F5: 산출물 손상** | 파일이 존재하나 내용이 불완전/깨짐 | 합본 중 일부 챕터 누락 |

---

## 2. 복구 프로토콜

### F1: 필수 파일 부재

```
감지: 템플릿의 "읽어야 할 파일"에 🔴 표시된 파일이 없음
복구:
  1. STATE.md 확인 → 해당 Phase가 완료 상태인지 체크
  2. 완료인데 파일 없음 → STATE.md 불일치 → P10 패턴으로 STATE 수정 후 해당 Phase 재실행
  3. 미완료 → "Phase {N} 미완료. 먼저 실행합니다." 안내 후 해당 Phase부터 재개
  4. 절대 금지: 빈 파일 생성하여 우회, 파일 없이 추측으로 작성
```

### F2: 에이전트 중단

```
감지: 에이전트가 산출물을 완성하지 못한 채 종료
복구:
  1. 부분 산출물 존재 여부 확인
  2. 부분 산출물 있음 → 마지막 완성된 섹션까지 보존 + 미완성 부분만 재실행
  3. 부분 산출물 없음 → 에이전트 재실행 (동일 프롬프트)
  4. 재실행 시 컨텍스트 최적화:
     - 불필요한 파일 참조 제거
     - 대용량 파일은 필요 섹션만 발췌
     - 산출물을 2개 이상 파일로 분할 고려
```

### F3: 검증 반복 실패

```
감지: 자동 수정 3회 후에도 동일 패턴 FAIL
복구:
  1. FAIL 원인 분류:
     a. 앵커 수치 자체가 비현실적 → S급 에스컬레이션 (사용자에게 수치 재확인)
     b. 산술 구조 문제 → 해당 챕터 전체 재작성 (부분 수정 아닌 초기화)
     c. 교차 문서 순환 충돌 → _contract.md를 유일 앵커로 강제 후 전 문서 갱신
  2. 사용자 보고: "검증 패턴 {P-ID}가 3회 수정 후에도 FAIL. 원인: {분석}. 제안: {A/B안}"
  3. 절대 금지: FAIL 무시하고 다음 Phase 진행
```

### F4: S급 결정 미응답

```
감지: S급 결정이 미확정인 상태에서 의존 Phase 진행 시도
복구:
  1. STATE.md 또는 context.md에서 미확정 S급 목록 추출
  2. 해당 S급에 의존하지 않는 병렬 작업은 진행 가능
  3. 의존하는 작업은 차단 → 사용자에게 리마인드
  4. 대기 중 진행 가능한 작업 목록 안내
```

### F5: 산출물 손상

```
감지: 파일 존재하나 구조가 불완전 (필수 섹션 누락, 마크다운 깨짐 등)
복구:
  1. 손상 범위 판단:
     a. 특정 섹션만 누락 → 해당 섹션만 재생성 (나머지 보존)
     b. 구조적 손상 (헤더 체계 깨짐) → 전체 재생성
  2. 재생성 전 백업: {파일명}.bak 으로 기존 파일 보존
  3. STATE.md에 "Phase {N}: 재생성 ({사유})" 기록
```

## 3. 자동 복구 매트릭스

| Phase | 가능한 장애 | 자동 복구 | 사용자 개입 필요 |
|-------|-----------|:---------:|:--------------:|
| Seed | F4 | - | ✅ (최초 인풋) |
| Research | F2 | ✅ 재실행 | - |
| Discovery | F1, F2, F4 | F1: Phase 복귀, F2: 재실행 | F4: S급 결정 |
| Skeleton | F1, F2 | ✅ | - |
| Planning | F1, F2, F3 | F1/F2: ✅, F3: 부분 | F3: 앵커 수치 확인 |
| Verify | F3, F5 | F5: ✅ | F3: 에스컬레이션 |
| Design | F1, F2 | ✅ | - |
| Implementation | F1, F2 | ✅ | - |
| Build~Deploy | F1, F2 | ✅ | - |

```markdown
## 복구 로그
| 일시 | Phase | 장애 유형 | 원인 | 조치 | 결과 |
|------|-------|----------|------|------|------|
| {날짜} | Planning | F3 | P1 수치 불일치 3회 | _contract.md 앵커 기준 ch07 재작성 | PASS |
```
